State 3 ways in which your site is accessible.
  1. Responsive Design:

    Implementing responsive design ensures that a website adjusts and displays appropriately across various devices and screen sizes (desktops, tablets, smartphones). This is crucial for users with different devices and abilities to access and navigate the website seamlessly.

  2. Accessibility Features and Compliance:

    Incorporating accessibility features and adhering to accessibility standards such as the Web Content Accessibility Guidelines (WCAG) helps make the website usable for individuals with disabilities. This can include providing alt text for images, ensuring proper keyboard navigation, using ARIA labels, and maintaining color contrast for those with visual impairments.

  3. Inclusive User Interface and User Experience (UI/UX):

    Designing an inclusive UI/UX involves considering the needs and preferences of diverse users. This may include providing options for adjusting text size, choosing color themes (dark mode or light mode), and enabling language preferences. An inclusive UI/UX ensures that users with different abilities and needs can customize the interface to enhance their user experience.

State 3 things you had to learn/find out by yourself when creating your site, and describe how you did that
  1. Using of Bootstrap

    Bootstrap is an incredibly powerful framework, especially when it comes to creating responsive websites. It offers a rich library of amazing styles that are not only ready to use but also highly customizable to suit your specific design needs.

  2. Fetching data from an API

    Utilizing an API as a data source simplifies the process of obtaining precisely what you require. This, in turn, facilitates the creation of your own website, minimizing the need to invest significant effort in building a custom database or backend project from scratch.

  3. Handlebars Tool

    When working with Handlebars, mastering the creation and utilization of partials is crucial for maintaining consistency throughout your project. Partial templates allow you to encapsulate reusable components, making it efficient to manage and apply consistent designs across various sections of your project. This approach streamlines development and ensures a cohesive and polished user experience.

What aspect(s) of your work do you think worked well and why?
  1. Handlebars

    Handlebars simplifies dynamic data display on websites by creating reusable templates with data placeholders. It separates data and presentation, enhancing code cleanliness and maintainability. This templating engine streamlines content rendering, making it effortless to display information dynamically, whether it's based on user input or database queries.

  2. Show news in list or grid

    With the capabilities of Handlebars, I've achieved a dynamic implementation allowing for the creation of multiple content containers to showcase news. This adaptable setup seamlessly facilitates the transition between distinct layouts, be it a list or grid.

  3. Sameler news

    Implementing a feature to gather and display condensed news articles from the currently selected category. Achieved through effective utilization of JavaScript to streamline and present the latest updates within the desired category."
